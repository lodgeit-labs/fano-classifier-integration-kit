⚠ CORS posture (read first)

Fano-engine at the production URL above does not currently emit CORS headers. A browser hitting it directly from a different origin will fail at the OPTIONS preflight (HTTP 405). This is the same Layer-7 consumer-class enforcement gap that surfaced for clawdog-calculator-api at mc12-2026-06-24 (L#66 CANDIDATE in Brain canon).

To run this demo today, you have three options:

  1. Run it from the same origin as Fano — not possible while Fano is on Cloud Run unless you proxy.
  2. Local proxy — run a tiny CORS-permissive proxy on your dev machine; point Base URL at http://localhost:8787; have it forward to Fano. See examples/demo-gui/PROXY.md.
  3. Disable browser CORS — for example launch Chrome with --disable-web-security --user-data-dir=/tmp/chrome-cors. Local-only experimentation; never use this profile for general browsing.

The Brain-canon next-sprint surface is to add CORSMiddleware at fano-engine/api/main.py + redeploy via the Streamace + canary + atomic-flip ladder (mirror of the Phase 4c sprint we just shipped). Once that lands, this demo runs natively against production with no proxy.
